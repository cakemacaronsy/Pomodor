# AGENTS.md

## 專案目的

這個專案使用番茄時鐘作為 vibe coding 的教學範例。重點不只是做出一個計時器，而是讓學生透過比較同一個 app 的多個版本，理解 app 客製化可以怎麼進行。

學生應該能夠在保留番茄時鐘核心邏輯的前提下，調整顏色、素材、文案、版面和小型互動行為。

## 預期版本結構

每一個客製化版本都應該是獨立版本，放在自己的資料夾，並且有自己的 HTML 入口檔。

| 版本 | 資料夾 | 入口 HTML | 用途 |
| --- | --- | --- | --- |
| Red | `versions/pomodoro-red/` | `index.html` | 基準紅色番茄時鐘版本 |
| Green | `versions/pomodoro-green_v1/` | `index.html` | 綠色自然風客製化版本 |
| Blue | `versions/pomodoro-blue_v1/` | `index.html` | 藍色冷靜、生產力風格客製化版本 |
| Red Strong | `versions/pomodoro-red-strong_v1/` | `index.html` | 更強烈視覺方向的紅色版本 |

根目錄可以保留捷徑檔，例如 `red.html` 或 `green.html`，但教學用的主要來源應該是 `versions/` 底下的版本資料夾。

## 客製化規則

- 實驗時不要覆蓋已經可用的版本。請先複製既有版本資料夾，再重新命名。
- 每個版本應該盡量自包含：HTML、CSS、JS 和 assets 應該放在一起，或清楚標示共用素材的引用方式。
- 除非課程目標就是修改功能，否則保留相同核心功能：專注計時、短休息、長休息、重設、開始 / 暫停、任務清單、可見狀態。
- 客製化差異要足夠明顯，方便學生比較：顏色 token、圖示 / 吉祥物、標題、按鈕樣式、計時器形狀、背景處理。
- 優先使用可讀、直接的程式碼，不要為了漂亮而做太抽象的架構。這是一個教學專案。

## 檔案命名

新增教學版本時，使用這個模式：

```text
versions/pomodoro-{theme}_v1/
  index.html
  styles.css
  app.js
  assets/
  VERSION.md
```

如果某個版本也需要放在根目錄作為捷徑，根目錄檔案只作為快捷入口。範例：`red.html` 可以指向紅色版本專用檔案，但 canonical red version 應該仍保留在 `versions/pomodoro-red/`。

## 教學註記

修改任何版本時，要讓改動容易被解釋：

1. 改了哪個視覺或功能概念？
2. 改了哪些檔案？
3. 哪些 CSS 變數或元件控制這個改動？
4. 學生要如何在另一個版本重複同樣的客製化？

課堂 demo 時，小而清楚的可見改動通常比看不見的重構更有效。

## 驗證

每次修改後：

- 直接用瀏覽器打開被修改的 HTML，或執行 `python3 -m http.server 4173`。
- 檢查桌面版和手機寬度。
- 確認計時器可以開始、暫停、重設，且模式切換正常。
- 確認每個版本都有載入自己的 CSS、JS 和 assets。
- 修改 JavaScript 時，執行 `node --check app.js` 或對應的 JS 檔案。

## 安全規則

- 除非 Sean 明確要求，不要刪除既有版本。
- 除非課程目標是講抽象化，否則不要把不同顏色版本合併成同一個共用 app。
- 如果同一個版本同時存在根目錄捷徑和版本資料夾，請保持兩邊同步。
